-Tu tarea NO es diseñar la web final.

- Debes seguir este marco conceptual:

#PARADIGMA:
- Action-First Fluid Design
- Mobile First extremo
- Conversión directa
- Diseño con profundidad (no plano)
- Estética Liquid Glass moderna (solo como referencia visual, no diseño final)

# IMPORTANTE:
- No inventes métricas.
- Si falta información crítica, pregúntala antes de generar el wireframe.
- No uses datos ficticios como seguidores, clientes o reviews.

FASE 2 — GENERAR WIREFRAME VISUAL

Genera una representación visual estructurada tipo Canva usando bloques claramente delimitados.

Debe incluir:

[ SECCIÓN HERO ]
- Eyebrow (Etiqueta confianza)
- H1 (5-7 palabras: Beneficio final + Cómo lo logra)
- TLDR breve (1 línea máx 80 caracteres)
- Botón CTA primario
- Microcopy mata-objeciones
- Espacio reservado para mockup visual (rectángulo placeholder)

[ SECCIÓN BENEFICIOS ]
- 3 a 5 bloques según plan
- Ícono placeholder
- Título corto
- 2 líneas descriptivas

[ SECCIÓN DIFERENCIADOR ]
- Tarjeta destacada
- Subtítulo fuerte
- Texto breve persuasivo

[ SECCIÓN CONFIANZA ]
- Espacio para prueba social (sin inventar números)
- Si no hay datos, usar “Espacio para testimonios”

[ SECCIÓN CTA FINAL ]
- Título directo
- Botón grande
- Recordatorio de beneficio
